# Phase 3 Plan 01: Live re:Fee E2E Preparation

## Goal

Prepare a safe, repeatable local workflow for the Phase 3 live USDT-TRC20 sweep
test without changing production application code.

## Scope

Create planning-only artifacts:

- `03-LIVE-E2E-RUNBOOK.md` - operator checklist for the live sweep.
- `phase3_refee_e2e.py` - helper for generating a sidecar-controlled wallet,
  checking readiness, and invoking `transfer_trc20_from`.

## Design

The helper defaults to an isolated DB under `/tmp/tron-shkeeper-phase3-e2e`.
This avoids mixing live e2e test keys with the repository's default `data/`
databases. It builds `REFEE` config from `/tmp/refee-live.env` variables when
`REFEE` is not already set, initializes the app, inserts `current_server_id=0`,
and uses existing app code for wallet generation and sweep execution.

## Acceptance Criteria

- The helper has an offline `self-test` command.
- `show-env` prints no API key and confirms re:Fee mode.
- `generate-wallet` creates a `keys.type='onetime'` wallet controlled by the
  sidecar and prints the address to fund.
- `check-wallet` reports private-key presence, account activation, USDT balance,
  TRX balance, and readiness for a clean zero-TRX sweep.
- `run-sweep --yes` refuses unsafe runs unless readiness criteria pass or the
  operator explicitly overrides with `--force`.

## Out Of Scope

- Running the actual USDT sweep before the generated address is funded.
- Writing production operator docs before Phase 3 has tx hashes and logs.
- Testing insufficient-funds refund behavior; the funded live order succeeded.
